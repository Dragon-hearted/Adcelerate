# Adcelerate Design System

> AI-Powered Marketing & Media Platform — a monorepo powering image generation, video pipelines, storyboards, and scraping systems, orchestrated through Claude Code skills, agents, and commands.

## Sources

This design system was reverse-engineered from:

- **Repo:** `Dragon-hearted/Adcelerate` (private; branch `master`) — the monorepo.
- **Client app (primary UI):** `apps/client/` — Vue 3 + Vite + TypeScript + Tailwind observability dashboard ("Multi-Agent Observability").
- **Theme system:** `apps/client/src/styles/themes.css` (12 predefined themes) and `apps/client/src/composables/useThemes.ts` (theme management, contrast validation, custom theme CRUD).
- **Brand marks:** `images/hero.svg` and `images/platform-overview.svg` — animated SVGs generated by `ReadmeEngine` with indigo-glow hub/spoke motif.
- **Uploaded asset:** `uploads/pixel_avatar.png` — pixel-art avatar used as a personal mark.
- **Tooling:** Bun runtime, TypeScript 5.8, Vue 3, Tailwind, Remotion 4, Hono 4, Zod 4, Playwright.

Copy-voice context was drawn from the monorepo README, the 34-skill library catalog (`library.yaml`), and the agent descriptions in `README.md` — all terse, emoji-tagged, mission-bullet style.

## The Platform

Adcelerate is a **hub-and-spoke** platform. One orchestrator (Adcelerate) coordinates six independent systems:

| System | What it does |
|---|---|
| **autoCaption** | Whisper.cpp + Remotion → TikTok-style word-highlighted captions on vertical video. |
| **SceneBoard** | CLI that turns a brief into a storyboard with scripts, timestamps, voice scripts, and NanoBanana prompts. |
| **Pinboard** | Terminal-first (Ink TUI) reference board + AI image generator. |
| **Instagram Scrapper** | Playwright + Instagram Private API — posts, reels, profile data. |
| **ImageEngine** | Centralized NanoBanana generation service with rate limits, budget guards, cost tracking. |
| **ReadmeEngine** | Auto-generates the repo-wide READMEs, animated SVG diagrams, and system docs. |

The **client** (what this DS recreates) is the observability dashboard for the agent orchestration layer — live websocket feed of hook events, filter panel, live pulse chart, HITL (human-in-the-loop) interactions, and 12-theme manager.

## Content Fundamentals

**Voice.** Terse, technical, imperative. Skills describe *when the user wants to X* — every one of the 34 skill descriptions in `library.yaml` begins with this clause. Agents get a one-sentence role plus a *when to use it* trailer. Copy reads like good CLI `--help` output: no marketing fluff, no exclamation points, no "we" or "you" — it's all **third-person task framing**.

**Casing.** Skill IDs are `kebab-case` (`cold-email`, `ai-seo`, `churn-prevention`). System names are CamelCase (`SceneBoard`, `ReadmeEngine`, `ImageEngine`, `AutoCaption`) and treated as proper nouns. UI labels are sentence-case (`Multi-Agent Observability`, `Connected`, `Clear events`). Status badges and event types are `PascalCase` (`PreToolUse`, `PostToolUse`, `SubagentStart`, `UserPromptSubmit`).

**Emoji are load-bearing.** Not decoration — actual information. README section headers lead with a single topical glyph (📦 Systems, 🏗 Architecture, 🛠 Tech Stack, 📚 Library, 🚀 Getting Started, 📂 Project Structure, 📝 Recent Changes, 🤝 Contributing, 📄 License). Event rows encode **event-type × tool-name** as an emoji pair (🔧 Bash → 🔧+💻, ✅ Read → ✅+📖). See `apps/client/src/composables/useEventEmojis.ts` for the full mapping.

**Vibe.** *Engineering-first with a pixel-art wink.* The README badge row (TypeScript/Bun/MIT) and the animated SVG hero set the technical tone; the pixel avatar and the uneven corners of the `🧡` sign-off ("Built with 🧡 using Bun, TypeScript, and Claude Code") say this is made by one person who likes their tools.

**Examples.**

- Skill (from `library.yaml`): *"When the user wants to write, rewrite, or improve marketing copy for any page — including homepage, landing pages, pricing pages, feature pages, about pages, or product pages."*
- Agent: *"Independent validation reviewer that checks output against soft acceptance criteria with a fresh context window. Produces structured validation reports."*
- UI label: `Connection lost · Attempting to reconnect...`
- Status pill: `PreToolUse`, `PostToolUseFailure`, `SubagentStart`
- Button: `Clear`, `Filters`, `Theme`, `Submit`, `Approve`, `Deny`

## Visual Foundations

### Palette

Twelve themes ship (`light`, `dark`, `modern`, `earth`, `glass`, `high-contrast`, `dark-blue`, `colorblind-friendly`, `ocean`, `midnight-purple`, `sunset-orange`, `mint-fresh`) — all share the same **token shape**: 4 primary steps, 4 bg steps, 4 text steps, 3 border steps, 4 semantic accents (success/warning/error/info), shadow × 2, hover/active/focus. The dashboard lives in **`dark`** by default; the brand mark lives in **`midnight-purple`** (matches hero SVG indigo `#6366F1` glow).

Source-app colors are **content-derived, not picked**: `useEventColors.ts` hashes the app name and returns `hsl(hash % 360, 70%, 50%)`, so every agent/system gets a stable but arbitrary hue.

### Typography

**Inter** for everything except code (`JetBrains Mono` as a reasonable substitute — the codebase doesn't ship a monospace, it uses `font-mono` CSS default). Weights used: 400, 500, 600, 700. The header title is `text-lg font-semibold` (18px / 600). Tabular nums are applied to counts and timestamps (`tabular-nums` class).

> ⚠️ **Font substitution flag.** Inter is used throughout and ships via Google Fonts CDN — no custom font files exist in the repo. `JetBrains Mono` is a substitution for the generic `font-mono` fallback; if you have a preferred mono, drop it in.

### Spacing, radii, shadows

- **Spacing:** Tailwind 4px base — density is *tight*. Most rows use `gap-1.5` (6px) or `gap-2` (8px); padding on cards is `p-3` (12px) or `p-4` (16px).
- **Radii:** `rounded-md` (6px) on buttons/badges/inputs, `rounded-lg` (8px) on cards and panels, `rounded-full` for dot indicators and pill badges.
- **Shadows:** Subtle — `0 1px 3px var(--theme-shadow)` on the header, `0 10px 15px var(--theme-shadow-lg)` on modals. Heavy drop shadows are avoided; emphasis comes from colored rings (`ring-1 ring-[var(--theme-accent-warning)]/20`) and tinted left-border accents on event rows.

### Borders & accents

Left-border color accents are used **sparingly and meaningfully**: the event row has a 4px `w-1` colored strip keyed to source-app hue; unresolved HITL cards get a `border-l-4` warning-colored border that flips to success when resolved. Don't slap colored left borders on cards that don't need state encoding.

### Transparency

Tint-over-surface is the pattern: `bg-[var(--theme-primary)]/10`, `bg-[var(--theme-accent-success)]/15`, `border-[var(--theme-accent-error)]/30`. Status pills, focus states, and tool-name chips all use this 8–15% alpha fill + 20–40% alpha border recipe over the base surface.

The `glass` theme uses real backdrop-blur (`.theme-glass .backdrop-blur { backdrop-filter: blur(10px); }`) — only invoked when explicitly opted in.

### Motion

Everything transitions on `var(--theme-transition)` = `all 0.2s ease-in-out` (or the faster `0.1s` variant). Vue `<Transition>` groups use `duration-200 ease-out` enter / `duration-150 ease-in` leave with `opacity` + `translate-y-2`/`max-h-0` pairings. Pulse dots animate with Tailwind's `animate-ping`. Spinners use `animate-spin`. Hero/platform SVGs ship their own SMIL animations (opacity fade-in + dash-offset flow).

### Hover / press states

- **Hover:** background steps up one tier (`bg-[var(--theme-bg-secondary)]` → `bg-[var(--theme-bg-tertiary)]`), text darkens (`text-secondary` → `text-primary`). No scale. No shadow change.
- **Press / active:** `bg-[var(--theme-active-bg)]` — same style, one more tier darker.
- **Focus:** 2px outline with `var(--theme-focus-ring)` at 2px offset.
- **Disabled:** `disabled:opacity-40 disabled:cursor-not-allowed`.

### Imagery & illustration

- **Animated SVG diagrams** are the hero imagery — hub-and-spoke, glowing rects with dashed arrows (see `assets/platform-overview.svg`). `ReadmeEngine` generates these.
- **Pixel-art avatar** (`assets/pixel_avatar.png`) — desk scene, warm bulb lighting, blue monitor glow, hoodie. Personal mark; the brand has a handmade, indie feel.
- No photography. No 3D renders. No full-bleed imagery in the dashboard. Backgrounds are flat surface colors.

### Layout

- **Fixed header** (`short:hidden` under 400px height) with logo · title · connection badge · toolbar.
- **Mobile breakpoint** is custom at **699px** (`'mobile': {'max': '699px'}`). Below that, labels collapse to icons and padding drops a tier.
- Dashboard is **top-down stacked**: header → optional filters → live pulse chart → optional swim lanes → infinite-scroll event timeline → floating "stick to bottom" button.
- Density is high — event rows are **compact**, timestamps are `text-xs`, multiple badges per row.

## Iconography

See the `ICONOGRAPHY` section below.

### Approach

Two parallel icon systems coexist:

1. **Inline Heroicons-style SVGs** for UI chrome — header icons, buttons, status. These are hand-written `<svg>` with `stroke-width="1.5"` or `"2"`, `viewBox="0 0 24 24"`, `fill="none" stroke="currentColor"`. Sized `w-4 h-4` (16px) to `w-5 h-5` (20px), shrinking to `w-3.5 h-3.5` on mobile. Examples shipped in the codebase: clear (trash), filters (funnel), theme (palette/swatch), warning (triangle-exclamation), check, chevron, user-group.
2. **Emoji as semantic glyphs** for events and tools — see `apps/client/src/composables/useEventEmojis.ts`. `PreToolUse: 🔧`, `PostToolUse: ✅`, `Bash: 💻`, `Read: 📖`, `Task: 🤖`, `Skill: ⚡`. These render in the timeline chips, chat transcript tool-use blocks, and live pulse chart legend.

No icon font. No SVG sprite. No CDN library. Icons live in-component, one copy per component — replace them by copying/pasting.

**CDN substitution flag:** If you need more coverage than the ~15 icons the codebase has, **Heroicons (outline, `stroke-width=1.5`)** is the closest match:

```html
<script type="module" src="https://unpkg.com/heroicons@2.2.0/24/outline/index.js"></script>
```

Stroke weight, 24×24 viewBox, and `currentColor` conventions all line up.

### Emoji usage

**Load-bearing, not decorative.** Use them where the codebase does:

- README section headers (one glyph per heading).
- Event type chips in the timeline (combo format: `eventEmoji+toolEmoji`).
- Copy-success confirmation (`✅`) and copy-failure (`❌`) button states.
- Tool-use block headers in chat transcript (`🔧 {tool_name}`).
- Sign-off line (`Built with 🧡 using Bun...`).

Don't scatter emoji into marketing copy or button labels. Don't use decorative emoji for "fun."

### Unicode

Not used as icons. Buttons are always text or SVG + text.
