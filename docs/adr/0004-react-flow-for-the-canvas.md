# Render the Canvas with React Flow (`@xyflow/react`)

The Canvas's defining behavior is a **live graph**: Steps stream in as a Run executes, wired into a Step Graph (a DAG), with Branches forming a Lineage tree, provenance edges, and a Stale cascade flowing downstream. Nodes appear *programmatically*, not by a human drawing them, and must render **heterogeneous rich cards** (prose, typed `Beat[]`-style tables, and images — confirmed from scene-board's real outputs), not just thumbnails. The web frontend is greenfield for this (React 19 / Next 15 / Tailwind / Zustand; today it renders only a flat linear timeline, no canvas or graph lib). We choose **React Flow (`@xyflow/react`)**: a custom node is just a React component (directly solving the rich-card requirement), and pan/zoom, edges, connection handles, and selection-box come built in. It pairs natively with Zustand (already in use) and is MIT-licensed. Auto-layout is added as needed via `dagre`/`elkjs` (React Flow positions nodes but does not lay out a DAG itself).

**Considered and rejected:**
- **tldraw** — best-in-class *freeform whiteboard* SDK, but the wrong category: its native model is freeform shapes, not connected nodes with auto-layout, so representing a live-generated DAG with lineage edges fights the grain. Also, its SDK is free only with a persistent "made with tldraw" watermark; removing it requires a paid business license. Defensible only if the Canvas were a freeform board first and the graph secondary — it is not (the user's mental model is "watching a graph build itself and branching from nodes").
- **Excalidraw** — same whiteboard mismatch; more an app than a clean embeddable graph SDK.
- **Custom (Konva / SVG + `d3-zoom` + `dagre`/`elk`)** — total control, zero lock-in, but hand-builds pan/zoom, hit-testing, edge routing, and selection — essentially rebuilding React Flow. Reconsider only if we outgrow what React Flow allows.

**Consequence:** The Canvas's node/edge data model becomes coupled to React Flow's — a meaningful but bounded lock-in (custom nodes are ordinary React, so the *card* rendering is portable; the graph plumbing is not). The **Spawn Tree** (left-zone agent telemetry, ADR-scope of the Console) deliberately does **not** use React Flow — it is a plain nested/collapsible React component with per-node counters, since it is a simple tree, not an editable graph.
