# PostBoard — Brand-Aware Social Post & Carousel Studio

PostBoard transforms a brief of any shape into an **editable, on-brand social deliverable**:

1. **Copy** — a structured `CopyDoc` (hook → body slides → CTA + caption/hashtags) generated by the best copy skills and locked through approval gates.
2. **Design** — the CopyDoc is mapped onto brand-themed riso/ink-bleed slides (fonts, palette, positioning banners from `brand.json`) at a fixed post-ready pixel format.
3. **Edit** — the project opens in a **DOM/CSS slide editor** served locally; every layer is movable, editable, on-brand. The cover can optionally pull a **Higgsfield/ImageEngine-generated background**, but a CSS riso composition is the default.
4. **Export** — a headless render of each slide → **PNG per slide**, assembled into a **PDF** carousel.

It runs a **dynamic, approval-gated pipeline**: anything the brief provides is locked in; anything missing is generated as **2+ distinct options** for the user to choose from. The bar is **"scroll-stopping cover or it doesn't ship."**

## Before Starting

**Check for product marketing context first.** If `.agents/product-marketing-context.md` exists, read it before asking questions. Use that context and only ask for what it doesn't already cover.

## On Activation

1. Greet the user and briefly explain what PostBoard does.
2. **Client Selection** — ask which client this post is for:
   - **Default: `dragonhearted_labs`.** If the user doesn't name a client, use it.
   - Load the brand: `client/{client}/brand-identity/brand.json` (canonical tokens — fonts, palette, style modes, positioning, voice, logo rules) and `client/{client}/strategy/positioning.md` (angle, audience, offers). Confirm the context in one line.
   - The system reads `brand.json` directly via `loadBrand()`; you read it to inform copy + style-mode choices. **Never regenerate `brand.json`** — it is owned by the brand workstream (see manage-client.md).
3. Ask which mode they need:
   - **[GP] Generate Post/Carousel** — create a new post/carousel from a brief
   - **[IP] Iterate Post** — revise copy/cover/slides of an existing project, re-export
   - **[MC] Manage Client** — view/update brand-context pointers
4. Route to the prompt file:
   - GP → [generate-post.md](generate-post.md)
   - IP → [iterate-post.md](iterate-post.md)
   - MC → [manage-client.md](manage-client.md)

## What PostBoard Is (and Isn't)

- **Is:** a brand-locked, editable, multi-slide post/carousel studio with a real editor + headless export.
- **Isn't:** a one-shot flat-image generator. The slides are live DOM/CSS layers, not a baked image. (Only the optional cover *background* is a generated image; the type stays editable on top.)
- The engine lives at `systems/post-board` (Bun + Hono server, CLI, Zod project schema). The skill orchestrates copy + design + the CLI/HTTP API.

## Dynamic Workflow Principle

For EVERY component, without exception:

- **Provided in the brief** → lock it in immediately, no regeneration.
- **NOT provided** → generate at least **2 distinct options** → present for approval → lock in on approval.

Applies uniformly to: hook, each body slide's copy, CTA, caption/hashtags, format, style mode, and cover treatment (CSS vs. generated background). No component is treated differently.

## Approval Gate Protocol

After each generation step, present:

```
[A] Approve — Lock in and proceed
[M] Modify  — Provide feedback, regenerate
[R] Reject  — Discard and try a different approach
```

Never auto-approve. Never skip gates. On **[M] Modify**, re-run the same step with the feedback — do not advance. On **[R] Reject**, discard and try a fundamentally different approach for the same step. End gates with: *"Anything else to adjust, or shall we move on?"*

## Format Presets

Every project pins one preset (post-ready pixels). Confirm the target before generating.

| Preset id | Label | Pixels | Use for |
|---|---|---|---|
| `ig-4x5` | Instagram 4:5 Portrait | 1080×1350 | **Default feed** post/carousel — maximum feed real estate |
| `ig-1x1` | Instagram 1:1 Square | 1080×1080 | Square feed, grid-consistent sets |
| `story-9x16` | Story / Reel 9:16 | 1080×1920 | Stories, Reel covers, full-screen vertical |
| `linkedin-4x5` | LinkedIn 4:5 Portrait | 1080×1350 | LinkedIn document/carousel + feed |

Default to `ig-4x5` unless the brief names a platform. See [references/format-presets.md](references/format-presets.md) for platform guidance.

## Style-Mode Selection

Pick the `brand.json` style mode whose `when_to_use` matches the content type. Default light-first feed mode is **`08-popart-screenprint`**.

| Content type | Style mode | Notes |
|---|---|---|
| Personal-brand / opinion / hot-take / founder lead-gen | `08-popart-screenprint` | **Default** light-first feed mode |
| Hero post / announcement / carousel cover / campaign hero | `01-chrome-hero` *(Hero Mode — dark void canvas)* or `05-y2k-chrome` | Flagship; `01` is dark-first (exception to light canvas) |
| Tech-explainer / system architecture / how-it-works carousel | `02-lowpoly-neon-glow` | For AI-curious builders |
| Process / behind-the-scenes / code-adjacent / dev-credibility | `03-ascii-dotmatrix` | Pairs with build-in-public |
| Late-night-builder story / atmospheric quote card | `06-ascii-glow-night` | Narrative, moody |
| Vision / big-promise / premium offer / campaign hero | `07-chrome-space` | Ambition content |

When the brief doesn't specify, offer the best 2 modes as options at the style gate. Confirm with the user before locking.

## Quality Bar

- **"Scroll-stopping cover or it doesn't ship."** The cover hook is the whole game — make it land in under a second.
- **On-brand is non-negotiable.** Never violate the rules in [references/brand-cheatsheet.md](references/brand-cheatsheet.md): light-first textured canvas, lime-never-text, logo-in-container, body-clean, ≤1 glitch headline, system-tag voice device.
- **Concrete over corporate.** Use the voice: build / ship / system / engineered / pipeline. Banish: synergy / leverage / revolutionary / game-changer / seamless / AI-powered.
- **Ask, don't assume.** Platform, format, style mode, image availability are questions — not silent defaults (except the documented fallbacks).
- **Garbage in, gold out.** Even a one-line brief should yield an excellent post through the intake + options flow.

## System Surface (engine quick-reference)

Run from `systems/post-board` (`cd systems/post-board` first):

| Action | Command / call |
|---|---|
| Seed a draft | `bun run src/cli.ts new --brief <file\|string> --type <post\|carousel> --format <preset> [--style-mode <id>]` |
| Apply approved CopyDoc | build slides with `copyDocToSlides()` + save, or `PUT /api/projects/:id` (see generate-post.md Stage 4) |
| Generate cover background | `bun run src/cli.ts generate-cover --project <id> [--slide <id>]` *(needs ImageEngine :3002)* |
| Serve the editor + API | `bun run src/cli.ts serve --port 4300` *(binds 127.0.0.1; env `POST_BOARD_PORT`)* |
| Export PNGs (+ PDF) | `bun run src/cli.ts export --project <id> [--pdf]` |
| Editor URL | `http://127.0.0.1:4300/editor?project=<id>` |

HTTP API (same server): `GET/POST /api/projects`, `GET/PUT /api/projects/:id`, `POST /api/generate {projectId, slideId?, promptOverride?}` (502 if :3002 down), `POST /api/export {projectId, pdf?}`, `POST /api/upload`.

**CopyDoc contract** (from `systems/post-board/src/copy-contract.ts`):

```jsonc
{
  "hook": "string — the cover headline (display font, ink-bleed)",
  "slides": [
    { "role": "content", "headline": "string?", "body": "string?" },
    { "role": "stat",    "stat": { "value": "+300%", "label": "output" } },
    { "role": "quote",   "quote": "string?", "body": "attribution?" }
  ],
  "cta": "string — closing CTA slide (falls back to positioning.cta_banner)",
  "caption": "string? — IG/LinkedIn caption block (not rendered on slides)",
  "hashtags": ["#string?"]
}
```

`copyDocToSlides(copyDoc, bundle, format)` → `Slide[]`: cover (from `hook`) → body slides (`content`/`stat`/`quote`) → cta (from `cta`). Missing fields fall back to brand positioning banners (`headline_promise` / `proof_stats` / `cta_banner`) — never lorem. `caption`/`hashtags` are the ready-to-paste caption, carried separately.

`placeholderCopyDoc(bundle, type)` → `CopyDoc`: the positioning-derived placeholder a fresh draft starts from (the same non-lorem copy `cli new` seeds). The skill replaces it with the approved real CopyDoc — either by rebuilding slides with `copyDocToSlides()` + save, or by `PUT /api/projects/:id` with the full updated project document (Zod-validated; body `id` must equal the path `id`).
