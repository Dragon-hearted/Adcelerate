# PostBoard â€” Brand-Aware Social Post & Carousel Studio

PostBoard transforms a brief of any shape into an **editable, on-brand social deliverable**:

1. **Copy** â€” a structured `CopyDoc` (hook â†’ body slides â†’ CTA + caption/hashtags) generated by the best copy skills and locked through approval gates.
2. **Design** â€” the CopyDoc is mapped through a **layout-variant registry + a deterministic carousel-arc picker** (`src/templates`) onto brand-themed riso/ink-bleed slides at a fixed post-ready pixel format. Every role has several **named layouts** (cover: `hero-editorial`/`split-accent`/`stacked-index`/`chrome-hero`; content: `hero-annotated`/`prompt-proof`/`grid-showcase`/`big-number`/`left-rule`; + stat/quote/cta), so **adjacent slides differ in their dominant device** â€” the deck reads as designed, not templated. Layouts compose **Neon-Lime accent blocks** (lime is a *surface* behind ink type, never lime text) and **brand graphic-element cutouts** (starburst / barcode / halftone / wireframe).
3. **Edit** â€” the project opens in a **DOM/CSS slide editor** served locally; every layer (text / image / element cutout / logo / shape accent) is movable, editable, on-brand. Carousels are **image-forward** by default: each slide is built around a **generated hero image** (Higgsfield CLI + **NanoBanana Pro**, the new image-engine default) with brand-locked text overlaid â€” but a CSS riso composition is the graceful fallback whenever generation is declined or ImageEngine is down.
4. **Export** â€” a headless render of each slide â†’ **PNG per slide**, assembled into a **PDF** carousel.

It runs a **dynamic, approval-gated pipeline**: anything the brief provides is locked in; anything missing is generated as **2+ distinct options** for the user to choose from. The bar is **"scroll-stopping, not templated"** â€” a cover that earns the tap and a deck with real layout rhythm.

## What PostBoard Is (and Isn't)

- **Is:** a brand-locked, editable, multi-slide post/carousel studio with a real editor + headless export.
- **Isn't:** a one-shot flat-image generator. The slides are live DOM/CSS layers, not a baked image. (Only the optional cover *background* is a generated image; the type stays editable on top.)
- The engine lives at `systems/post-board` (Bun + Hono server, CLI, Zod project schema). The skill orchestrates copy + design + the CLI/HTTP API.

## Dynamic Workflow Principle

For EVERY component, without exception:

- **Provided in the brief** â†’ lock it in immediately, no regeneration.
- **NOT provided** â†’ generate at least **2 distinct options** â†’ present for approval â†’ lock in on approval.

Applies uniformly to: hook, each body slide's copy, CTA, caption/hashtags, format, style mode, and cover/hero treatment (CSS riso vs. generated NanoBanana heroes). No component is treated differently.

## Approval Gate Protocol

After each generation step, present:

```
[A] Approve â€” Lock in and proceed
[M] Modify  â€” Provide feedback, regenerate
[R] Reject  â€” Discard and try a different approach
```

Never auto-approve. Never skip gates. On **[M] Modify**, re-run the same step with the feedback â€” do not advance. On **[R] Reject**, discard and try a fundamentally different approach for the same step. End gates with: *"Anything else to adjust, or shall we move on?"*

## Layout System â€” Variants, Arc, Accents, Cutouts

The slide geometry is no longer one fixed stack per role. `copyDocToSlides()` and `cli new` both route every slide through the **layout-variant registry** (`src/templates/variants.ts`) and a **deterministic carousel-arc picker** (`src/templates/arc.ts`, seeded off the project id â€” never random, so re-seeding is stable). You don't pick variants by hand for the copy path; you write copy that **feeds the arc**, and the editor's "add slide" menu exposes the variants for manual placement.

- **Variant registry (image-forward).** Cover: `hero-editorial` (default), `split-accent`, `stacked-index`, `chrome-hero`. Content: `hero-annotated`, `prompt-proof`, `grid-showcase`, `big-number`, `left-rule`. Stat: `stat-block` / `stat-hero`. Quote: `quote-bleed` / `quote-raster`. CTA: `cta-save` (default â€” "save this post") / `cta-hero`. Each reserves a **hero-image zone** for the generated image; text frames it.
- **Carousel arc (reference-derived, task #4).** Canonical 7-slide spine: `cover (hook) â†’ content (method) â†’ content (prompt-proof) â†’ stat (proof) â†’ content (range) â†’ quote (reset) â†’ cta (save)`. The picker guarantees **adjacent slides never share a dominant device**, consecutive `content` slides rotate, and the deck holds ONE ground colour throughout â€” variety comes from hero treatment, not per-slide ground shuffling. See `client/dragonhearted_labs/strategy/reference/ohneis652-carousel-patterns.md`.
- **Neon-Lime accent blocks.** The brand's "pop" device: a lime (`#C6FF00`) marker-block **surface** with Graphite ink type on top â€” `cta-save` and the first `stat-block` always carry one. **Lime is never a text colour** on the light canvas (~1.1:1); the renderer enforces it. Represented as a `shape` layer (the new accent/rule layer kind).
- **Brand cutout elements (task #2).** Transparent per-mark cutouts (`starburst` 1â€“3 as stat punctuation, `barcode` â‰¤1 as a corner authenticity stamp, `halftone` as a â‰¤15% background field, `wireframe-globe` as a low-opacity motif) compose as `element` layers and are draggable from the editor palette. Keep the element budget calm: body/"how" slides stay quietest.

To write copy that exploits this: give the deck a clear hook â†’ method â†’ proof â†’ range â†’ reset â†’ save progression, supply a real **stat** (so the lime `stat-block` lands) and a punchy **quote** (so the reset beat hits), and keep one idea per slide.

## Quality Bar

- **"Scroll-stopping, not templated."** The cover hook is the whole game â€” make it land in under a second â€” AND the deck must show **visible layout variety**: adjacent slides differ in dominant device, with â‰Ą1 Neon-Lime accent block and â‰Ą1 brand graphic-element in the deck. A uniform kickerâ†’headlineâ†’body stack is a fail. (Soft bar: `knowledge/acceptance-criteria.md`.)
- **On-brand is non-negotiable.** Never violate the rules in [references/brand-cheatsheet.md](references/brand-cheatsheet.md): light-first textured canvas, lime-never-text, logo-in-container, body-clean, â‰¤1 glitch headline, system-tag voice device.
- **Concrete over corporate.** Use the voice: build / ship / system / engineered / pipeline. Banish: synergy / leverage / revolutionary / game-changer / seamless / AI-powered.
- **Ask, don't assume.** Platform, format, style mode, image availability are questions â€” not silent defaults (except the documented fallbacks).
- **Garbage in, gold out.** Even a one-line brief should yield an excellent post through the intake + options flow.

## System Surface (engine quick-reference)

Run from `systems/post-board` (`cd systems/post-board` first):

| Action | Command / call |
|---|---|
| Seed a draft | `bun run src/cli.ts new --brief <file\|string> --type <post\|carousel> --format <preset> [--style-mode <id>]` |
| Apply approved CopyDoc | build slides with `copyDocToSlides()` + save, or `PUT /api/projects/:id` (see generate-post.md Stage 4) |
| Generate cover background | `bun run src/cli.ts generate-cover --project <id> [--slide <id>]` *(needs ImageEngine :3002)* |
| **Generate hero per slide** | `bun run src/cli.ts generate-heroes --project <id> [--slide <id>â€¦] [--allow-fallback]` *(image-forward; batch via :3002)* |
| Serve the editor + API | `bun run src/cli.ts serve --port 4300` *(binds 127.0.0.1; env `POST_BOARD_PORT`)* |
| Export PNGs (+ PDF) | `bun run src/cli.ts export --project <id> [--pdf]` |
| Editor URL | `http://127.0.0.1:4300/editor?project=<id>` |

HTTP API (same server): `GET/POST /api/projects`, `GET/PUT /api/projects/:id`, `POST /api/generate {projectId, slideId?, promptOverride?}` (502 if :3002 down), `POST /api/generate-heroes {projectId, slideIds?, autoFallback?}` (batch per-slide heroes; 502 on engine failure), `POST /api/export {projectId, pdf?}`, `POST /api/upload`.

**Image-forward by default.** Seeding is fast and image-free but **image-forward**: every slide carries an editable `heroPrompt` (the intended NanoBanana hero, derived from the slide's copy + style mode + brand `background_system`) and a hero-image-zone layout, with the **CSS-riso background as the fallback shown until a hero is generated**. One `generate-heroes` pass fills them; the brand-locked text stays editable on top. In the editor, each slide has a **"âś¦ Generate hero (NanoBanana)"** affordance (and a "revert to CSS background").

**Image generation default & fallback.** PostBoard omits the `model`, so ImageEngine applies its default â€” the **Higgsfield CLI + NanoBanana Pro** (`higgsfield-nano-banana-pro`). The old silent gemini fallback is now **permission-gated**: with NanoBanana down and no model, ImageEngine returns a clear error (cover-gen / hero-gen surface it; the CSS-riso background keeps working) rather than silently swapping providers. The operator approves a switch via `--allow-fallback` (CLI), `autoFallback: true` (API), or `IMAGE_ENGINE_AUTO_FALLBACK=1` (env). Per-slide hero failures are graceful â€” that slide just keeps its CSS-riso background.

**CopyDoc contract** (from `systems/post-board/src/copy-contract.ts`):

```jsonc
{
  "hook": "string â€” the cover headline (display font, ink-bleed)",
  "slides": [
    { "role": "content", "headline": "string?", "body": "string?" },
    { "role": "stat",    "stat": { "value": "+300%", "label": "output" } },
    { "role": "quote",   "quote": "string?", "body": "attribution?" }
  ],
  "cta": "string â€” closing CTA slide (falls back to positioning.cta_banner)",
  "caption": "string? â€” IG/LinkedIn caption block (not rendered on slides)",
  "hashtags": ["#string?"]
}
```

`copyDocToSlides(copyDoc, bundle, format)` â†’ `Slide[]`: cover (from `hook`) â†’ body slides (`content`/`stat`/`quote`) â†’ cta (from `cta`). Missing fields fall back to brand positioning banners (`headline_promise` / `proof_stats` / `cta_banner`) â€” never lorem. `caption`/`hashtags` are the ready-to-paste caption, carried separately.

`placeholderCopyDoc(bundle, type)` â†’ `CopyDoc`: the positioning-derived placeholder a fresh draft starts from (the same non-lorem copy `cli new` seeds). The skill replaces it with the approved real CopyDoc â€” either by rebuilding slides with `copyDocToSlides()` + save, or by `PUT /api/projects/:id` with the full updated project document (Zod-validated; body `id` must equal the path `id`).
